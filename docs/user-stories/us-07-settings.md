# US-07：持续维护中心

**模块**：Settings
**对应 PRD**：`docs/prd-v2.md §2.1 配置中心`、`docs/prd-v2.1.md`

## 模块定位

`US-07` 在 v1 中不负责首次成功主路径，而是负责：

- 维护首次成功后已经生效的关键配置
- 提供高级入口与系统维护能力
- 在不打断默认模板主路径的前提下，为进阶用户提供可见的控制点

默认原则：

- 首次成功走 `Onboarding + 模板工坊`
- 持续维护走 `Settings`
- 高级系统能力从这里后置展开

---

## 一、关键持续维护能力

### US-07-01：维护 LLM Provider 与 Key

**作为** 已开始使用的用户，
**我希望** 在设置中查看、更新和切换 LLM 配置，
**以便** 我在不重走首次向导的情况下继续使用助手。

**优先级**：P0
**验收标准（AC）**：

- AC1：Settings 提供独立的 LLM 配置入口
- AC2：用户可以更新 API Key、切换 Provider 或调整默认模型
- AC3：已有配置应以安全方式展示，如遮码
- AC4：变更后不会让用户重新理解底层系统结构

---

### US-07-02：维护 Gateway 的高级参数

**作为** 进阶用户，
**我希望** 在设置中调整 Gateway 的高级参数，
**以便** 在需要时进行更深的系统控制。

**优先级**：P1
**验收标准（AC）**：

- AC1：Gateway 设置存在于 Settings，而不是首次成功默认主路径
- AC2：支持调整端口、认证模式等高级项
- AC3：保存后清楚提示是否需要重启才能生效

---

### US-07-03：重新运行首次成功向导

**作为** 用户，
**我希望** 在设置中重新运行首次成功向导，
**以便** 当我的关键配置出问题时，可以按引导重新恢复可用状态。

**优先级**：P0
**验收标准（AC）**：

- AC1：Settings 提供“重新运行首次成功向导”入口
- AC2：重新运行时优先复用已有配置
- AC3：重新运行的目标是恢复可用助手，而不是单纯回到配置控制台

---

## 二、后置高级能力

### US-07-04：管理 Channel 白名单

**作为** 进阶用户，
**我希望** 在设置中管理可接入的 Channel，
**以便** 控制访问来源。

**优先级**：P1
**验收标准（AC）**：

- AC1：支持启用 / 禁用 Channel
- AC2：支持查看和复制 Token
- AC3：支持添加自定义 Channel

---

### US-07-05：管理 Agent 配置入口

**作为** 进阶用户，
**我希望** 在设置中访问 Agent 管理能力，
**以便** 在需要时处理本地运行实体，而不是在首次成功路径里先理解它。

**优先级**：P1
**验收标准（AC）**：

- AC1：Settings 提供统一的 Agent 管理入口
- AC2：该入口默认不干扰模板创建与首次使用主路径
- AC3：具体 Agent 能力定义以 `US-09` 为准

---

### US-07-06：导出与恢复配置备份

**作为** 高频使用者，
**我希望** 导出与恢复关键配置，
**以便** 在迁移设备或恢复环境时降低成本。

**优先级**：P2
**验收标准（AC）**：

- AC1：支持导出关键配置
- AC2：导入时校验格式和版本
- AC3：涉及敏感信息时要求重新补录

---

### US-07-07：高级能力统一后置

**作为** 产品设计者，
**我希望** 将 Channel、Agent、Hooks 等能力统一放入设置或高级入口，
**以便** 非技术用户不会在默认主路径中被控制台心智打断。

**优先级**：P1
**验收标准（AC）**：

- AC1：高级系统能力不作为首页或首次向导的默认前台信息
- AC2：用户需要时可以在设置中找到它们
- AC3：高级能力入口文案应强调“可选”“高级”而不是“必配”
