# US-07：配置中心

**模块**：Settings
**对应 PRD**：`docs/prd-v2.md §2.1 配置中心`、`docs/prd-v2.1.md`

---

### US-07-01：管理 LLM Provider

**作为** AI 开发者，
**我希望** 在配置中心随时添加、修改或删除 LLM Provider，
**以便** 灵活切换不同的模型服务。

**优先级**：P0
**验收标准（AC）**：

- AC1：Provider 列表显示已配置的所有内置和自定义 Provider
- AC2：支持新增、编辑（复用配置向导的表单 UI）和删除
- AC3：删除前检查是否有 Bot 正在使用该 Provider
- AC4：API Key 展示为遮码形式，点击图标复制

---

### US-07-02：修改 Gateway 参数

**作为** AI 开发者，
**我希望** 在配置中心修改 Gateway 的端口、认证模式等参数，
**以便** 在首次配置后仍可按需调整。

**优先级**：P0
**验收标准（AC）**：

- AC1：配置界面与配置向导 step1 保持一致的表单风格
- AC2：保存后提示是否需要重启 Gateway 以生效
- AC3：一键触发 Gateway 重启

---

### US-07-03：管理 Channel 白名单

**作为** AI 开发者，
**我希望** 在配置中心管理哪些客户端（lysmata 桌面端、VS Code 插件等）可以接入 Gateway，
**以便** 控制访问权限。

**优先级**：P1
**验收标准（AC）**：

- AC1：Channel 列表以 Toggle 行形式展示，可逐项启用/禁用
- AC2：可查看和复制每个 Channel 的 Token
- AC3：支持添加自定义 Channel 并生成随机 Token

---

### US-07-04：重新运行配置向导

**作为** 用户，
**我希望** 在配置中心找到"重新运行向导"入口，
**以便** 按引导式流程重新完成所有配置，而不是逐项手动修改。

**优先级**：P1
**验收标准（AC）**：

- AC1：Settings 页有"重新运行配置向导"按钮
- AC2：点击后打开 Onboarding 向导，直接从 step1 开始
- AC3：完成向导后返回主界面

---

### US-07-06：Agent 配置管理入口  ⚠️ 待补充（见 US-09）

**作为** AI 开发者，
**我希望** 在配置中心直接访问 Agent 配置管理功能（查看 Agent 列表、新增、绑定、移除），
**以便** 无需离开 Lysmata 即可完成 OpenClaw Agent 的全生命周期管理。

**优先级**：P0
**实现状态**：⚠️ 未实现（完整功能定义见 US-09）
**验收标准（AC）**：

- AC1：Settings 页新增「Agent 管理」Tab（或独立区块），作为 US-09 所有功能的统一入口
- AC2：该 Tab 内嵌完整的 Agent 列表（US-09-01）、新增（US-09-02）、绑定（US-09-03）、移除（US-09-04）、绑定关系总览（US-09-05）和非交互式批量配置（US-09-06）功能
- AC3：Settings 页原有的「重新运行配置向导」按钮（US-07-04）在重进向导时，若检测到本地已有 Agent，在「创建 Agent」步骤（US-01-03b）自动预填已有 Agent 信息

---

### US-07-05：导出与恢复配置备份

**作为** 研究者，
**我希望** 将当前所有配置导出为备份文件，并能在新环境中一键恢复，
**以便** 迁移设备或灾难恢复。

**优先级**：P2
**验收标准（AC）**：

- AC1：导出生成一个加密的配置包文件（不含明文 API Key）
- AC2：导入时验证文件格式和版本兼容性
- AC3：导入成功后提示需要重新输入 API Key
