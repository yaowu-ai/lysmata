# US-02：助手工坊 & Bot 管理

**模块**：Assistant Workshop & Bot Management
**对应设计**：`design/ui-bot-management.html`、`docs/prd-v2.md §2.5`

## 模块定位

`US-02` 在 v1 中分成两层：

1. **前台主路径**：模板创建助手，帮助用户快速得到第一个可用助手
2. **后台高级路径**：Bot 管理与编辑，服务进阶用户与持续维护

因此，v1 默认不再让用户从空白表单开始“手工建 Bot”，而是优先通过模板实例化助手。

## 术语说明

- `助手`：用户能理解、能直接开始使用的前台对象
- `Bot`：Lysmata 应用层的连接与配置对象，是助手在系统内的载体
- `模板`：帮助用户快速创建助手的入口，不要求用户先理解底层连接结构

---

## 一、模板助手工坊（v1 主路径）

### US-02-01：首页展示 3 个模板助手入口

**作为** 首次使用的用户，
**我希望** 在首页直接看到 3 个清晰的模板卡片，
**以便** 我知道自己可以从哪个方向开始创建第一个助手。

**优先级**：P0
**验收标准（AC）**：

- AC1：首页第一主区展示 3 个模板：`通用助手`、`信息助手`、`日常任务助手`
- AC2：每个模板卡片包含名称、用途说明、创建按钮
- AC3：模板描述使用用户语言，不使用 Agent / Gateway 等底层概念作为卡片主文案
- AC4：模板卡片是首页的第一优先级信息，不被系统状态区抢占视觉中心

---

### US-02-02：通过模板快速创建第一个助手

**作为** 初次使用的用户，
**我希望** 点击模板后通过最少输入完成助手创建，
**以便** 我马上得到一个能用的助手，而不是先填写复杂配置表单。

**优先级**：P0
**验收标准（AC）**：

- AC1：点击模板后进入轻量创建流程，默认只要求极少输入项，如助手名称
- AC2：创建过程中自动复用已完成的 LLM 配置
- AC3：系统自动补齐默认 Bot、默认连接和必要后台对象
- AC4：成功后直接进入对应助手的对话页或唯一明确的开始使用入口

---

### US-02-03：模板创建流程支持轻量定制

**作为** 想稍微定制助手的用户，
**我希望** 在模板创建时能改少量可理解的参数，
**以便** 助手更符合我的用途，同时不被过多技术项打断。

**优先级**：P0
**验收标准（AC）**：

- AC1：模板创建阶段允许调整名称、头像、简短用途说明等轻量字段
- AC2：这些字段都应使用用户语言表达，不要求手工填写 Gateway 或 Agent
- AC3：高级配置不默认展开

---

### US-02-04：首页展示最近使用与继续使用

**作为** 已创建过助手的用户，
**我希望** 首页能看到最近使用的助手和继续使用入口，
**以便** 我不用每次都重新创建或重新寻找入口。

**优先级**：P0
**验收标准（AC）**：

- AC1：首页第二主区展示最近使用的助手卡片或最近会话
- AC2：每个助手卡片提供“继续使用”主操作
- AC3：空状态时优先回到模板引导，而不是展示复杂空白控制台

---

## 二、高级 Bot 管理（进阶路径）

### US-02-05：查看所有已创建助手 / Bot 列表

**作为** 进阶用户，
**我希望** 在管理页看到所有已创建助手对应的 Bot 列表，
**以便** 我了解当前有哪些对象正在被系统使用与维护。

**优先级**：P0
**验收标准（AC）**：

- AC1：列表按卡片形式展示已创建助手 / Bot
- AC2：卡片信息应优先展示用户能理解的名称、用途和状态
- AC3：底层连接信息可以展示，但应处于次级层级

---

### US-02-06：编辑助手的基础信息

**作为** 用户，
**我希望** 修改助手的名称、头像和用途描述，
**以便** 它更符合我的持续使用场景。

**优先级**：P0
**验收标准（AC）**：

- AC1：编辑入口允许修改名称、头像、用途描述和激活状态
- AC2：保存后卡片与相关会话视图自动更新
- AC3：基础编辑不要求理解高级连接项

---

### US-02-07：进入高级配置编辑 Bot

**作为** 进阶用户，
**我希望** 在需要时进入高级模式编辑 Bot 的连接、技能与扩展项，
**以便** 我在不影响默认主路径的前提下获得更强控制能力。

**优先级**：P1
**验收标准（AC）**：

- AC1：高级编辑入口与默认模板创建入口分离
- AC2：高级编辑中可继续保留原有的基础 / MCP / Skills / 连接等分组
- AC3：高级编辑不应作为首次创建助手的默认路径

---

### US-02-08：测试助手 / Bot 的连接状态

**作为** 用户，
**我希望** 对某个助手执行连接测试并看到结果，
**以便** 我知道它为什么能用或为什么不可用。

**优先级**：P0
**验收标准（AC）**：

- AC1：列表卡片和高级编辑页都提供测试连接入口
- AC2：测试中有明确 loading 状态
- AC3：成功时显示成功反馈与延迟等信息
- AC4：失败时显示用户可理解的错误原因

---

### US-02-09：查看运行状态与持续健康信息

**作为** 进阶用户，
**我希望** 查看单个助手 / Bot 的运行时状态，
**以便** 在需要时诊断连接和健康问题。

**优先级**：P1
**验收标准（AC）**：

- AC1：卡片上提供状态详情入口
- AC2：状态详情展示连接、健康、心跳和待处理状态
- AC3：状态变化应能被实时更新

---

### US-02-10：删除不再需要的助手 / Bot

**作为** 用户，
**我希望** 删除不再需要的助手，
**以便** 保持列表整洁，并避免无效对象继续占用心智空间。

**优先级**：P0
**验收标准（AC）**：

- AC1：删除前必须有确认
- AC2：若存在活跃会话，需要清楚提示删除影响
- AC3：删除成功后列表自动更新
