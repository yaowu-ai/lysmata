# US-02：Bot 管理

**模块**：Bot Management
**对应设计**：`design/ui-bot-management.html`、`docs/prd-v2.md §2.5`

---

### US-02-01：查看所有 Bot 列表

**作为** AI 开发者，
**我希望** 在主界面以卡片列表形式看到所有已创建的 Bot，
**以便** 快速掌握各 Bot 的状态和基本信息。

**优先级**：P0
**验收标准（AC）**：

- AC1：每张卡片显示 Bot 名称、Emoji 头像、描述和连接状态指示灯
- AC2：连接状态以四态指示（connected 绿 / disconnected 灰 / connecting 黄动画 / error 红）
- AC3：列表支持搜索过滤，按名称实时筛选

---

### US-02-02：创建新 Bot

**作为** AI 开发者，
**我希望** 通过右侧抽屉表单创建新的 Bot，
**以便** 连接不同的 OpenClaw Gateway 实例或配置专属技能组合。

**优先级**：P0
**验收标准（AC）**：

- AC1：点击"+ 新建 Bot"按钮，右侧滑入抽屉
- AC2：抽屉包含四个 Tab：基础信息 / Skills / MCP / 连接配置
- AC3：基础信息 Tab 含名称（必填）、Emoji 头像选择器、描述（可选）
- AC4：连接配置 Tab 含 Gateway URL、端口、认证 Token 输入框
- AC5：保存时校验必填字段，成功后列表刷新并 Toast 提示

---

### US-02-03：编辑已有 Bot

**作为** AI 开发者，
**我希望** 点击 Bot 卡片进入编辑状态，
**以便** 修改 Bot 的名称、描述、连接参数或技能配置。

**优先级**：P0
**验收标准（AC）**：

- AC1：点击卡片或卡片上的"编辑"按钮滑入相同的右侧抽屉，字段预填已有数据
- AC2：修改保存后卡片信息实时更新

---

### US-02-04：删除 Bot

**作为** AI 开发者，
**我希望** 删除不再需要的 Bot，
**以便** 保持列表整洁。

**优先级**：P0
**验收标准（AC）**：

- AC1：点击删除时弹出二次确认对话框
- AC2：若 Bot 有活跃会话，确认框额外提示"该 Bot 存在进行中的对话，删除后相关数据将保留"
- AC3：确认删除后从列表移除，后台断开 WebSocket 连接

---

### US-02-05：测试 Bot 连接

**作为** AI 开发者，
**我希望** 对每个 Bot 单独执行连接测试，
**以便** 确认 Gateway 地址和认证配置正确。

**优先级**：P0
**验收标准（AC）**：

- AC1：Bot 卡片或编辑抽屉内有"测试连接"按钮
- AC2：点击后按钮显示旋转 Spinner，状态变为 connecting
- AC3：连接成功时显示绿色"已连接"徽章和 RTT 延迟
- AC4：连接失败时显示红色错误信息和具体原因（超时 / 认证失败 / 地址不可达）

---

### US-02-06：实时监控 Bot 连接状态

**作为** 研究者，
**我希望** Bot 列表实时反映每个 Bot 的连接状态变化，
**以便** 不需要手动刷新即可感知断线或重连。

**优先级**：P1
**验收标准（AC）**：

- AC1：通过 SSE 全局流接收 Gateway 推送的 presence、health、heartbeat 事件
- AC2：状态变化时卡片指示灯无感刷新，connecting 状态以脉冲动画标识
- AC3：Bot 异常断线时系统托盘图标变化并发送桌面通知
