# US-01：安装向导 & 配置向导

**模块**：Onboarding
**对应设计**：`design/ui-onboarding.html`、`docs/prd-v2.1.md`

---

### US-01-01：零终端一键安装 OpenClaw

**作为** 非技术用户，
**我希望** 在 lysmata 应用内点击一个按钮即可完成 OpenClaw 的全部安装，
**以便** 不需要打开终端或手动执行任何命令。

**优先级**：P0
**验收标准（AC）**：

- AC1：欢迎页（Intro）显示"开始安装 OpenClaw"主按钮和"已安装 OpenClaw，直接配置 →"次按钮；Footer 显示"取消"按钮
- AC2：点击"开始安装 OpenClaw"后跳转至"环境检测"页，Footer 变为"上一步"+"一键安装"
- AC3：点击"一键安装"后跳转至"正在安装"页，实时显示进度条（百分比）和终端日志滚动区（JetBrains Mono 字体、深色背景）；Footer 按钮变为禁用状态
- AC4：安装成功后自动跳转至"安装成功"页，显示"立即配置 →"（主按钮）和"稍后再说，先去主界面"（次按钮）；Footer 隐藏"取消"按钮
- AC5：安装失败时，"正在安装"页给出明确错误信息，并提供重试按钮
- AC6：全程无需用户输入终端命令

---

### US-01-02：环境预检与状态反馈

**作为** AI 开发者，
**我希望** 在安装前看到系统环境（Node.js、权限、网络）的检测结果，
**以便** 提前发现问题，避免安装中途失败。

**优先级**：P0
**验收标准（AC）**：

- AC1：进入"环境检测"页时，自动并发扫描：Node.js 版本（≥ v18.0）、系统权限（守护进程执行权限）、网络连通性（连接至 OpenClaw Registry）
- AC2：每项检测用绿/黄/红状态灯直观呈现（绿：通过，黄：检测中/警告，红：失败）；正在检测的条目显示呼吸闪烁动画
- AC3：检测失败的项目给出具体原因和修复建议文字
- AC4：全部检测通过后，Footer 的"一键安装"按钮才可点击；存在红色失败项时按钮保持禁用

### US-01-04：可视化 Gateway 配置

**作为** AI 开发者，
**我希望** 通过表单界面配置 Gateway 端口、绑定地址、认证模式和开机自启，
**以便** 不需要手动编辑 JSON 配置文件。

**优先级**：P0
**验收标准（AC）**：

- AC1：表单提供绑定地址（默认 127.0.0.1）和监听端口（默认 18789）两栏并排输入；每字段下方显示内联说明文字
- AC2：认证模式以 `<select>` 下拉呈现，选项为 "None（本地无感，推荐）" 和 "Token（需鉴权）"
- AC3：开机自启通过 Toggle 开关设置，视觉上显示滑动圆点，默认开启（蓝色）
- AC4：提交时执行 POST `/openclaw/gateway-config` 写入配置，成功后进入 step2
- AC5：**若 Gateway 已有配置（配置文件中已存在相关字段），页面加载时通过 `GET /settings/gateway` 获取当前值并预填至表单**，而非使用硬编码默认值；用户可在此基础上修改

### US-01-05：可视化 LLM Provider 配置

**作为** AI 开发者，
**我希望** 通过卡片选择内置 Provider 并填写 API Key，或通过表单自定义第三方 Provider，
**以便** 无需了解底层 JSON schema 即可激活大模型服务。

**优先级**：P0
**验收标准（AC）**：

- AC1：内置 Provider（OpenAI、Anthropic、Groq、Moonshot）以 4 列卡片网格展示；点击卡片高亮选中，下方显示对应的 API Key 密码输入框
- AC2：API Key 使用 `type="password"` 输入；提交时通过 PUT `/settings/llm` 写入，不上传云端
- AC3："自定义 Provider" Tab 提供 Provider ID、显示名称、Base URL、Model ID、API Key、API 类型（下拉）等字段；提供 Ollama / vLLM / LM Studio / Moonshot 一键模板快填按钮
- AC4：必填字段（Provider ID、Base URL、Model ID）未填时，点击"下一步"显示红色边框高亮对应输入框并弹出错误 Toast
- AC5："Marketplace 🛒" Tab 提供商业增值路径入口（购买额度一键激活）
- AC6：**若已有 LLM 配置，页面加载时通过 `GET /settings/llm` 获取当前值并预填**：内置 Provider 场景自动选中对应 Provider 卡片，并在 API Key 输入框显示已配置提示（掩码占位，不明文回显）；自定义 Provider 场景切换至对应 Tab 并预填各字段

### US-01-06：跳过可选配置步骤

**作为** 希望快速上手的用户，
**我希望** 在 Channel、Skills、Hooks 等可选步骤点击"跳过此步"，
**以便** 先用最基础的配置启动，后续再按需补充。

**优先级**：P1
**验收标准（AC）**：

- AC1：step3（Channel）、step4（Skills）、step5（Hooks）的步骤徽章（badge）显示"可跳过"文字，Footer 右侧显示"跳过此步"链接按钮
- AC2：点击"跳过此步"后，顶部进度器（Stepper）该步骤圆圈显示"–"并变为灰色，连接线变为灰色虚线；同时弹出 Toast："已跳过「{步骤名}」，可在设置中随时配置"
- AC3：step6（Review）汇总显示已跳过的步骤列表（黄色警示框），提示可在设置中补充
- AC4：step1（Gateway）和 step2（Provider）不显示"跳过此步"选项，步骤徽章标注"必填"

### US-01-07：配置完成后重新运行向导

**作为** 需要重新配置的用户，
**我希望** 在完成页或主界面触发重新运行配置向导，
**以便** 修改已有配置或重新走一遍初始化流程。

**优先级**：P1
**验收标准（AC）**：

- AC1：完成页（Done）显示"重新运行配置向导"链接按钮（带刷新图标）
- AC2：配置向导 Header 右上角有"✕"退出按钮；点击后执行 markOnboardingComplete()，弹出 Toast 提示"已退出向导，配置将保留已完成步骤"，然后跳转至 Bot 管理主界面
- AC3：点击"重新运行配置向导"时，清空 skippedSteps 记录，从 step1 重新开始（不重复安装流程）
- AC4：**重进向导时，step1 和 step2 的表单通过接口读取当前系统配置值预填，而非展示硬编码默认值**，方便用户在原有配置基础上修改

### US-01-09：已配置场景回填现有配置值

**作为** 已完成过初始化配置的用户，
**我希望** 重新进入配置向导时，各配置步骤自动读取并展示当前系统的实际配置值，
**以便** 我能清晰看到现有配置，并在其基础上按需修改，而不是每次都从默认值重头填写。

**优先级**：P0
**验收标准（AC）**：

- AC1：GatewayConfigView（step1）在挂载时调用 `GET /settings/gateway`；若接口返回成功，将返回的 `port`、`bindAddress`、`authMode`、`autostart` 值填入对应表单字段；接口失败或返回空时退化为硬编码默认值
- AC2：ProviderConfigView（step2）在挂载时调用 `GET /settings/llm`；若 `defaultModel.primary` 字段存在，则：匹配为内置 Provider 时自动选中对应 Provider 卡片，并在 API Key 输入框内显示掩码占位符（`•••••••••••`），不明文回显密钥；匹配为自定义 Provider 时切换至"自定义 Provider" Tab 并预填 Provider ID / Base URL / Model ID 等字段
- AC3：ReviewView（step6）在挂载时分别调用 `GET /settings/gateway` 和 `GET /settings/llm`，将实际已保存的值动态生成 diff 预览（格式：`+ key: value`），而非展示硬编码占位字符串
- AC4：上述接口调用在 loading 期间对应表单区域显示骨架屏或 loading 状态，接口成功返回后再渲染表单，避免表单闪烁或初始值错位
- AC5：重新提交时，若用户未修改某字段（如 API Key 掩码占位），该字段不覆盖后端已有值（通过检测掩码占位符标识来区分"未改动"与"已清空"两种意图）
