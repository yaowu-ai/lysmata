# US-03：首次成功对话

**模块**：Private Chat
**对应设计**：`design/ui-private-chat.html`、`docs/prd-v2.md §2.1`

## 模块定位

`US-03` 在 v1 中不是“普通聊天页”，而是 **首次成功的兑现位**。

也就是说，用户完成模板创建之后，必须在这里真正感受到：

- 助手已经可用
- 第一次对话是顺畅的
- 回答是可读的
- 我愿意继续用下去

因此 v1 对私聊的要求不是“功能全”，而是“第一轮真实使用成功”。

## v1 最小完成度

| 能力 | v1 要求 |
| --- | --- |
| 创建后立即进入会话 | 必须 |
| 流式输出 | 必须 |
| 停止生成 | 必须 |
| Markdown 渲染 | 必须 |
| 历史持久化 | 必须 |
| 删除确认与自动切换 | 应纳入 v1 |
| 搜索 / 分页 | v1.x |
| 右侧详情面板 | 后置 |

---

## 一、首次成功对话

### US-03-01：模板创建后直接进入第一次会话

**作为** 刚刚创建完助手的用户，
**我希望** 创建成功后直接进入第一次会话，
**以便** 我立刻验证这个助手真的能用。

**优先级**：P0
**验收标准（AC）**：

- AC1：模板创建成功后直接打开对应助手的对话页，或给出唯一明确的“立即开始对话”入口
- AC2：用户不需要先回到列表页再手动选择助手
- AC3：页面首屏明确显示当前助手是谁

---

### US-03-02：流式回复与可中断生成

**作为** 用户，
**我希望** 助手以流式方式输出回复，并在需要时停止生成，
**以便** 我获得实时反馈，同时保有控制感。

**优先级**：P0
**验收标准（AC）**：

- AC1：回复内容逐步流式显示
- AC2：生成期间发送按钮切换为“停止”或等效操作
- AC3：点击停止后当前生成被中断，并保留已生成内容
- AC4：生成完成后输入区恢复可继续提问状态

---

### US-03-03：Bot 回复默认支持 Markdown 渲染

**作为** 用户，
**我希望** 助手回复中的列表、代码、表格等内容是格式化可读的，
**以便** 我不需要阅读原始 Markdown 语法。

**优先级**：P0
**验收标准（AC）**：

- AC1：Bot 消息默认支持基础 Markdown 渲染
- AC2：代码块、行内代码、列表、标题、表格等常见元素可读
- AC3：流式输出过程中也能维持基本可读性
- AC4：用户消息保持纯文本显示

---

### US-03-04：历史消息自动持久化

**作为** 用户，
**我希望** 重新打开应用后仍能看到之前的对话，
**以便** 我把这个助手当作持续使用的工具，而不是一次性演示。

**优先级**：P0
**验收标准（AC）**：

- AC1：再次进入会话时自动加载历史消息
- AC2：消息顺序正确，历史不会丢失
- AC3：同一助手的连续使用路径是自然的

---

## 二、会话管理

### US-03-05：删除会话前明确确认，删除后自动切换

**作为** 用户，
**我希望** 删除会话前有确认，并在删除后自动切换到合理位置，
**以便** 不会因为误删或空白状态打断体验。

**优先级**：P0
**验收标准（AC）**：

- AC1：删除前必须确认
- AC2：如果删除的是当前会话，系统自动切换到最近的其他会话或合适空状态
- AC3：删除不应让页面进入令人困惑的空白状态

---

### US-03-06：查看历史会话列表并继续使用

**作为** 用户，
**我希望** 在侧边栏快速找到已有会话并继续使用，
**以便** 我能把助手当成长期入口，而不是每次重建。

**优先级**：P0
**验收标准（AC）**：

- AC1：会话列表按最近使用排序
- AC2：点击已有会话可继续对话
- AC3：空状态文案鼓励继续使用已有助手或返回创建入口

---

## 三、后置增强能力

### US-03-07：搜索历史会话

**作为** 高频用户，
**我希望** 通过关键词搜索历史会话，
**以便** 更快定位过去讨论过的内容。

**优先级**：P1
**验收标准（AC）**：

- AC1：支持按关键词过滤历史会话
- AC2：搜索结果高亮匹配内容

---

### US-03-08：历史消息分页

**作为** 高频用户，
**我希望** 在消息很多时按需加载更早历史，
**以便** 保持长会话的性能与可读性。

**优先级**：P1
**验收标准（AC）**：

- AC1：长会话支持分页或懒加载
- AC2：滚动体验自然，不影响当前阅读

---

### US-03-09：右侧助手详情面板

**作为** 进阶用户，
**我希望** 在聊天页查看当前助手的详细信息，
**以便** 理解它的配置、技能与状态。

**优先级**：P1
**验收标准（AC）**：

- AC1：可选显示右侧详情面板
- AC2：展示助手名称、用途、连接状态和关键摘要
- AC3：该面板不应影响 v1 首次成功主路径
