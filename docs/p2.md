# 重构 onboarding模块 

## 整体
- 把操作分成mac和window，按照 shared/os/index.ts 下面的策略去区分，让相关的业务代码只负责引用文件，不要老是在业务里面if else 判断
- 公共的及界面UI相关的直接放在页面中，不需要区分mac windows
- 整体onboarding操作分成4步，每步有子路由，整体界面顶上要有导航栏，展示当前在哪一步，哪些已完成，哪些未完成

## 步骤1 检查环境
- 这里做的事情是，检测用户的环境，并根据情况安装、更新或展示成功界面
- 以检测默认shell里面执行 openclaw --version 的输出的标识。有版本号 如【OpenClaw 2026.3.13 (61d171a)】表示安装，命令不成功表示未安装
- 已安装展示步骤 开始对话、进入配置
- 未安装展示步骤 安装OpenClaw

## 步骤2 安装OpenClaw
- 这里需要展示安装的进度条
- 如果需要管理员权限，在桌面应用里面提示输入密码或者用生物识别
