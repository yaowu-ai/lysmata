接下来，只需几个简单步骤，即可完成 Gateway 和大模型提供商的配置。

1. Gateway 配置

设置 OpenClaw Gateway 的基础运行参数。

本地使用通常保持 127.0.0.1，局域网共享可设为 0.0.0.0

默认 18789，若有冲突请修改

建议本地环境使用 None，提升开发体验

开机自启 (Daemon)
让 Gateway 作为后台服务随系统启动

3. Channel 配置

配置可接入到 Gateway 的客户端通道白名单与鉴权。

4. Skills 配置

从 ClawHub 安装或配置本地自定义技能。

🖥️
System Execution
允许大模型在本地执行终端命令和读写文件。
🌐
Web Search
允许大模型调用搜索引擎获取实时信息。

5. Hooks 配置

设置请求的预处理或后处理钩子逻辑。

全局日志拦截器
path: /hooks/global-logger.js

6. 检查与应用

确认配置变更，应用并重启 Gateway。
